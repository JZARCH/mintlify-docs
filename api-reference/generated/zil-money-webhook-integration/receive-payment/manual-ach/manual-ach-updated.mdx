## manual-ach.updated

Triggered whenever the **status** of a manual ACH pull transfer changes.

**Trigger:** The ACH transfer status is updated by the banking network (e.g., moves from `PENDING` → `PROCESSING` → `COMPLETED`, or `FAILED` / `RETURNED`).

**Source:** `BankingAchTransferRequestObserver::updated()` — fires only when `status` field changes.

---

## Status Transitions

```
PENDING → PROCESSING → COMPLETED
                    ↘ FAILED
COMPLETED → RETURNED  (ACH return received post-settlement)
PENDING   → CANCELED  (user canceled before processing)
```

## Payload Field Reference

Same schema as `manual-ach.initiated`. The `data.status` field will reflect the new status, and `data.updatedAt` will be updated.

| Field | Type | Description |
|-------|------|-------------|
| `data.status` | string | New status: `PENDING`, `PROCESSING`, `COMPLETED`, `CANCELED`, `FAILED`, `RETURNED` |
| `data.achPullReturnCode` | string\|null | Populated if status is `RETURNED` (e.g., `R01` = Insufficient Funds, `R02` = Account Closed) |
| `data.achPullReturnDescription` | string\|null | Human-readable return reason (populated with `achPullReturnCode`) |
