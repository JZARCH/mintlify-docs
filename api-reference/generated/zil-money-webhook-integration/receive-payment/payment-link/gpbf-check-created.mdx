## gpbf.check.created

Triggered when a customer submits a **Get Paid By Form** (GPBF) payment, creating a new eCheck/digital check record.

**Trigger:** A customer fills out and submits the payment form sent by a Zil Money user. A `GetPaidByFormCheck` record is created in the system.


---

## Payload Field Reference

| Field | Type | Description |
|-------|------|-------------|
| `data.referenceId` | string | Encrypted unique check reference ID |
| `data.memo` | string | Check memo / payment note |
| `data.amount` | string | Payment amount formatted to 2 decimal places |
| `data.serialNumber` | integer | Check serial number |
| `data.invoiceNumber` | string | Invoice number (empty string if not provided) |
| `data.transactionId` | string | Associated transaction ID (empty string if not set) |
| `data.customerDetails.customerEmail` | string | Payer's email address |
| `data.customerDetails.customerName` | string | Payer's full name |
| `data.customerDetails.customerAddressLine1` | string | Payer's street address |
| `data.customerDetails.customerPhone` | string | Payer's phone number |
| `data.customerDetails.customerCity` | string | Payer's city |
| `data.customerDetails.customerState` | string | Payer's state (2-letter code) |
| `data.customerDetails.customerZip` | string | Payer's ZIP code |
| `data.metaData` | array | Additional metadata attached to the payment form (empty array `[]` when not set) |
| `data.customField1` | string | Custom field 1 configured on the payment form |
| `data.customField2` | string | Custom field 2 |
| `data.customField3` | string | Custom field 3 |
| `data.status.statusCode` | integer | Numeric status code from `central_received_check_status` table (e.g., `1`) |
| `data.status.statusShortDescription` | string | Short description from DB (e.g., `"GPBL Acpt/Rej"`) |
| `data.createdAt` | ISO 8601 datetime | Check creation timestamp |

## HEADERS

| Header | Value |
| --- | --- |
| Content-Type | application/json |
| Signature | {{hmac_signature}}<br/>HMAC-SHA256 of the raw JSON request body, signed with your webhook_secret: hash_hmac('sha256', rawJsonBody, webhookSecret) |

## Example Request

```http
POST /webhooks/zilmoney HTTP/1.1
Host: your-server.com
Content-Type: application/json
Signature: {{hmac_signature}}
```
