## gpbf.check.updated

Triggered whenever the **status** of a GPBF eCheck/digital check payment changes.

**Trigger:** The received check status is updated in the system (e.g., payment clears, fails, or is returned). Fires only when the `status` field changes on a `ReceivedCheckList` record with `received_type = 4`.

**Source:** `ReceivedCheckUpdateWebHook::handle()` → `ReceivedCheckListResource`

---

Same schema as `gpbf.check.created`. Key fields that change on update:

| Field | Type | Description |
|-------|------|-------------|
| `data.status.statusCode` | integer | Updated numeric status code from `central_received_check_status` table |
| `data.status.statusShortDescription` | string | Updated short description matching the new status code |

## Notes on status fields

- `statusCode` is a **database-driven integer** — not a fixed enum. The value depends on the `central_received_check_status` table entries for `received_type = 4`
- `statusShortDescription` is the `status_short_description` column from the same table
- `metaData` is always an **array** (`[]` when empty, not an object `{}`)
- `serialNumber` is always an **integer**, not a string
