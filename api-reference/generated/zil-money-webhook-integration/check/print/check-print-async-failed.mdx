## check.print.async (failed)

Triggered when an **asynchronous check print job fails** — the PDF could not be generated.

**Trigger:** `AsynchronousCheckPrintJob` encounters an error (PDF generation failure, page count mismatch, or upstream error).


---

## Payload Field Reference

| Field | Type | Description |
|-------|------|-------------|
| `data.status` | string | `"failed"` — print job failed |
| `data.requestId` | string (UUID) | The async print request ID |
| `data.url` | string | Empty string `""` on failure |
| `data.checkIds` | array of strings | Encrypted IDs of the checks that failed to print |
| `data.errorMsg` | string | Human-readable error message (present on page count mismatch) |
| `data.msg` | string | Additional error detail from the PDF generation service |

## Status Values

| `data.status` | Meaning |
|---------------|---------|
| `success` | PDF generated — `url` is populated |
| `failed` | Print failed — `url` is empty, check `errorMsg` / `msg` |
