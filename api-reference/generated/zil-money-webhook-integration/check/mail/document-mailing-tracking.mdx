## document.mailing.tracking

Triggered each time a **carrier tracking update** is received for a mailed document. Fires multiple times as the document moves through the postal network.

**Trigger:** A new carrier tracking scan is recorded for a `BusinessToolsDocumentMailing` record.

**Source:** `SendCheckfloTrackingWebhookJob::handle()` → `getDocumentData()` (referance_type = 2)

---

## Payload Field Reference

| Field | Type | Description |
|-------|------|-------------|
| `data.id` | string | Encrypted document reference ID |
| `data.mode` | string | Environment: `production` or `local` |
| `data.carrier` | string | Carrier name: `USPS`, `UPS`, `FEDEX` |
| `data.mailType` | string | Mail class ID (e.g., `"1"` = USPS First Class) |
| `data.trackingUrl` | string | Public carrier tracking URL |
| `data.trackingCode` | string | Carrier tracking number |
| `data.trackingStatus` | string | Current tracking status (see below) |
| `data.estimatedDeliveryDate` | string | Estimated delivery date — empty string if unknown |
| `data.trackingStatusDesc` | string | Human-readable carrier status description |
| `data.updatedDate` | string | Timestamp of the tracking scan (format: `"Jan 17,2024 02:20:00 PM"`) |

> **Note:** Unlike `check.mailing.tracking`, document tracking events do **not** include a `groupId` field.

## Tracking Status Values

| Status | Description |
|--------|-------------|
| `ACCEPTED` | Accepted by carrier at facility |
| `PRE_TRANSIT` | Label created, not yet picked up |
| `IN_TRANSIT` | Package in transit between facilities |
| `OUT_FOR_DELIVERY` | Out for delivery to recipient |
| `DELIVERED` | Successfully delivered (`DE` status code) |
| `RETURNED` | Returned to sender (undeliverable) |
| `FAILURE` | Delivery failed or exception occurred |
