## card-program.card.transaction.updated

Triggered when a **card transaction status changes** — e.g., an authorization settles, is reversed, or is declined after initially pending.

**Trigger:** A `CardProgramCardTransaction` record is updated with a new status or reviewed state.

**Source:** `CardProgramCardTransaction::boot()::updated()` → `CardProgramCardTransactionResource`

---

Same schema as `card-program.card.transaction.created`. The `data.status` field reflects the new transaction state.

## Status Transitions

```
APPROVED → SETTLED       (authorization settles, typically T+1)
APPROVED → REVERSED      (merchant voids/reverses the transaction)
PENDING  → APPROVED      (authorization completes)
PENDING  → DECLINED      (authorization denied)
```
