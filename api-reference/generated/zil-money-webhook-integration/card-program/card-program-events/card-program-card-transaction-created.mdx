## card-program.card.transaction.created

Triggered when a **new card transaction** is recorded â€” typically when an authorization is received from the card network.

**Trigger:** A `CardProgramCardTransaction` record is created (e.g., cardholder makes a purchase).


---

## Payload Field Reference

| Field | Type | Description |
|-------|------|-------------|
| `data.id` | string (UUID) | Unique identifier for this transaction |
| `data.cardProgramIssuingRecordId` | string (UUID) | UUID of the issuing record (card program account) |
| `data.cardId` | string (UUID) | UUID of the card that processed the transaction |
| `data.amount` | string | Transaction amount |
| `data.description` | string | Transaction description |
| `data.status` | string | Transaction status: `APPROVED`, `DECLINED`, `PENDING`, `REVERSED`, `SETTLED` |
| `data.is_reviewed` | integer | `1` if reviewed by user, `0` if not |
| `data.transactionDate` | ISO 8601 datetime | Date and time the transaction occurred |
| `data.merchantName` | string | Merchant name (uses override name if set, otherwise original) |
| `data.merchantCity` | string | Merchant city |
| `data.merchantState` | string | Merchant state |
| `data.merchantCountry` | string | Merchant country code |
| `data.merchantCategoryCode` | string | 4-digit MCC (Merchant Category Code) |
| `data.merchantCategory` | string | Human-readable merchant category description |

## HEADERS

| Header | Value |
| --- | --- |
| Content-Type | application/json |
| Signature | {{hmac_signature}}<br/>HMAC-SHA256 of the raw JSON request body, signed with your webhook_secret: hash_hmac('sha256', rawJsonBody, webhookSecret) |

## Example Request

```http
POST /webhooks/zilmoney HTTP/1.1
Host: your-server.com
Content-Type: application/json
Signature: {{hmac_signature}}
```
