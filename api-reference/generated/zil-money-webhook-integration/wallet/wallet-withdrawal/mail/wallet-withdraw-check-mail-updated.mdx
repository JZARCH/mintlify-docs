## wallet.withdraw.check.mail.updated

Triggered whenever the **status** of a check mail withdrawal changes. This event fires **multiple times** across the check's lifecycle.

**Trigger conditions:**
1. Status changes (e.g., `PENDING` → `PROCESSING` → `COMPLETED`)
2. Tracking number assigned after check is handed to carrier
3. Clearing/preview files generated after printing

**Source:** `BankingCheckMailingRequest::boot()::updated()` → `WithdrawFundViaMailCheckResources`

---

## Lifecycle of updates

| Fire # | What changed | `status` | `trackingNumber` | `clearingFile` |
|--------|-------------|----------|-----------------|----------------|
| 1st | Status → `PROCESSING` | `PROCESSING` | `null` | `null` |
| 2nd | Check printed, files ready | `PROCESSING` | `null` | populated URL |
| 3rd | Handed to carrier | `PROCESSING` | populated | populated URL |
| 4th | Status → `COMPLETED` | `COMPLETED` | populated | populated URL |

> This sample shows the **first update** (status change only). Fields populate in later fires.

---

## Payload Field Reference

| Field | Type | Description |
|-------|------|-------------|
| `data.id` | string (UUID) | Unique identifier for this check mail request |
| `data.fromWallet` | string | Source wallet UUID |
| `data.amount` | string | Check amount |
| `data.description` | string | Memo / description |
| `data.shippingType` | string | Carrier and service (e.g., `FEDEX_NEXT_DAY_PM`, `USPS_FIRST_CLASS`) |
| `data.shippingInfo.trackingNumber` | string\|null | Carrier tracking number — `null` until handed to carrier |
| `data.shippingInfo.trackingUrl` | string\|null | Public tracking URL — `null` until handed to carrier |
| `data.status` | string | `PENDING`, `PROCESSING`, `COMPLETED`, `CANCELED`, `FAILED`, `RETURNED` |
| `data.clearingFile` | string\|null | Pre-signed S3 URL to clearing PDF — `null` until printed |
| `data.previewFile` | string\|null | Pre-signed S3 URL to check preview — `null` until printed |
| `data.reviewMessage` | string | Message if check is under review |
| `data.fromAddress` | object | Sender address (same structure as initiated) |
| `data.toAddress` | object | Recipient address (same structure as initiated) |
| `data.updatedAt` | ISO 8601 datetime | Timestamp of this update |
| `data.createdAt` | ISO 8601 datetime | Original creation timestamp |

> **Note:** Pre-signed S3 URLs for `clearingFile` and `previewFile` expire — download promptly after receiving.
