## wallet.funding.request.initiated

Triggered when a **wallet fund request is created** — a request for an external party to wire funds into a Zil Money wallet. The `depositInformation` object contains the beneficiary bank details that the depositor should use to complete the wire transfer.

**Trigger:** A fund request is created via the Zil Money platform or API.

---

## Payload Field Reference

| Field | Type | Description |
|-------|------|-------------|
| `data.id` | string (UUID) | Unique identifier for this fund request |
| `data.depositorName` | string | Name of the person/entity sending the funds |
| `data.toWallet` | string (UUID) | Destination wallet UUID |
| `data.amount` | string | Expected deposit amount (2 decimal places) |
| `data.description` | string | Fund request description / memo |
| `data.status` | string | `PENDING`, `COMPLETED`, `CANCELED`, `FAILED` |
| `data.transferType` | string | Transfer method — e.g., `"domestic-wire"` |
| `data.depositInformation.beneficiaryName` | string | Account holder name at the receiving bank |
| `data.depositInformation.beneficiaryRoutingNumber` | string | ABA routing number of the receiving bank |
| `data.depositInformation.beneficiaryAccountNumber` | string | Receiving bank account number |
| `data.depositInformation.beneficiaryAccountType` | string | Account type (e.g., `"Business Checking"`) |
| `data.depositInformation.beneficiaryBankAddress` | string | Physical address of the receiving bank |
| `data.depositInformation.beneficiaryBankSwift` | string | SWIFT/BIC code of the receiving bank |
| `data.depositInformation.specialInstruction` | string | Memo/reference the depositor must include in the wire |
| `data.updatedAt` | ISO 8601 datetime | Last updated timestamp |
| `data.createdAt` | ISO 8601 datetime | Creation timestamp |

> **Important:** Share the `depositInformation` fields with the depositor so they can correctly initiate the wire. The `specialInstruction` must be included in the wire memo to ensure the funds are credited to the correct wallet.

## HEADERS

| Header | Value |
| --- | --- |
| Content-Type | application/json |
| Signature | {{hmac_signature}}<br/>HMAC-SHA256 of the raw JSON request body, signed with your webhook_secret: hash_hmac('sha256', rawJsonBody, webhookSecret) |

## Example Request

```http
POST /webhooks/zilmoney HTTP/1.1
Host: your-server.com
Content-Type: application/json
Signature: {{hmac_signature}}
```
