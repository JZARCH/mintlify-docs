---
title: 'Create Virtual Card'
api: 'POST /card-program/card/create'
---

This API endpoint allows you to create a virtual or physical spend card linked to a specified wallet, with configurable spending limits and reset intervals for enhanced expense control.

**Payload Parameters**

| Parameter | Type | Required | Description |
| --- | --- | --- | --- |
| walletId | string | ✅ Yes | Unique identifier for the wallet to which the new card should be linked. |
| limitAmount | number | ✅ Yes | Maximum allowable spending amount on the card, per the defined limitInterval. |
| nameOnCard | string | ✅ Yes | nameOnCardThe name to be printed on the card |
| label | string | ✅ Yes | A user-defined label or nickname to help identify the card. |
| cardType | string | ✅ Yes | Type of the card being issued. Accepted values: virtual, physical. |
| limitInterval | string | ✅ Yes | Time interval for resetting the card limit. Accepted values: 'monthly', 'allTime', 'daily', 'weekly', 'perTransaction', 'yearly' |
| restrictionSettings | object | ⬜ Optional | Additional rules and controls to limit how and where the card can be used. |

**restrictionSettings Object**

| **Field** | **Type** | Description |
| --- | --- | --- |
| validTill | string (YYYY-MM-DD) | Expiration date for the card's usability. Transactions beyond this date will be declined. |
| allowedMerchantNames | array of strings | List of allowed merchant names where the card can be used. |
| allowedMerchantCategoryCode |  | List of allowed Merchant Category Codes (MCCs). Only merchants with these MCCs can accept payments from this card. |
| allowedMerchantCountry | array of strings | alpha-2 country codes. Card will be restricted to use in these countries. |
| allowedMerchantCity | array of strings | List of allowed cities for card usage. |
| allowedTime | object | Defines time-based usage restrictions for the card. |

```bash

  --header "Authorization: Bearer $AUTH_TOKEN" \
  --header "Accept: application/json"
```
